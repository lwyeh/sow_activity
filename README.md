## 系統簡介

本專案為荒野親子團設計，以 Google Apps Script 作為後端、Google Sheets 作為資料庫、HTML/CSS/JavaScript 作為前端介面，無需伺服器即可部署一套完整的活動報名與收費管理系統。

系統分為兩個入口：
- **一般頁面**（無參數）：供家長查詢並報名活動
- **管理員頁面**（`?page=admin`）：供幹部管理成員、活動、收費與出席統計

---

## 功能特色

- ✅ 活動列表顯示與開放/截止狀態判斷
- ✅ 家庭為單位的活動報名（出席 / 不出席 / 待定）
- ✅ 出席統計：依家庭、依團別、依小隊多維度彙整
- ✅ 收費活動管理：設備項目、數量上限、金額設定
- ✅ 繳費申報與管理員審核流程
- ✅ 成員資料管理（含自然名、年級、職位、隊名等）
- ✅ 家庭資料管理
- ✅ 管理員登入驗證（MD5 雜湊密碼保護）
- ✅ RWD 響應式設計，支援手機瀏覽
- ✅ 一鍵初始化/修復 Google Sheets 工作表結構

## 系統架構

### 檔案結構

```
sow_activity/
│
├── 後端（Google Apps Script）
│   ├── Main.gs              # 系統常數、網頁入口 doGet()、工作表初始化 setupSheets()
│   ├── Helpers.gs           # 共用工具函式（getSheet、genId、formatDate、getDb）
│   ├── Api_Activities.gs    # 活動管理 & 報名紀錄 & 出席統計 API
│   ├── Api_Members.gs       # 成員資料 & 家庭資料 CRUD API
│   ├── Api_Payment.gs       # 收費活動、收費明細、繳費紀錄 API
│   └── Api_System.gs        # 系統層 API（getAllData、管理員登入等）
│
└── 前端（HTML Templates）
    ├── Index.html           # 主頁面框架，根據 isAdminPage 決定載入內容
    ├── CSS_Main.html        # 全域 CSS 樣式
    ├── Modals.html          # 各功能彈窗 HTML 結構
    ├── JS_Main.html         # 主頁面 JavaScript（報名流程）
    ├── JS_RSVP.html         # 報名相關互動邏輯
    ├── JS_Admin.html        # 管理員後台 JavaScript
    └── JS_Payment.html      # 收費與繳費相關 JavaScript
```

### 各模組職責說明

| 檔案 | 層次 | 職責 |
|------|------|------|
| `Main.gs` | 後端 | 定義全域常數（工作表名稱、選單選項）、`doGet()` 入口、`setupSheets()` 初始化 |
| `Helpers.gs` | 後端 | 資料庫連線 `getDb()`、工作表存取 `getSheet()`、ID 產生、日期格式化 |
| `Api_Activities.gs` | 後端 | 活動 CRUD、報名紀錄讀寫 `submitRsvp()`、出席統計計算 `getActivityStats()` |
| `Api_Members.gs` | 後端 | 成員與家庭資料 CRUD |
| `Api_Payment.gs` | 後端 | 收費活動、設備項目、收費明細、繳費紀錄管理 |
| `Api_System.gs` | 後端 | 一次性打包所有資料 `getAllData()`、管理員登入驗證 |
| `Index.html` | 前端 | 主框架，透過 `<?= isAdminPage ?>` 切換一般/管理員模式 |
| `CSS_Main.html` | 前端 | 全站樣式表，RWD 設計 |
| `Modals.html` | 前端 | 新增/編輯成員、活動、收費等各式彈窗 HTML |
| `JS_Main.html` | 前端 | 頁面初始化、活動列表渲染、導航控制 |
| `JS_RSVP.html` | 前端 | 報名流程：載入家庭成員、選擇狀態、送出報名 |
| `JS_Admin.html` | 前端 | 管理員功能：成員/家庭/活動管理、出席統計表格 |
| `JS_Payment.html` | 前端 | 收費項目設定、繳費申報、審核流程 |

### 資料庫結構（Google Sheets）

| 工作表名稱 | 用途 | 主要欄位 |
|-----------|------|---------|
| `成員資料` | 所有團員資訊 | 成員ID、家庭ID、姓名、自然名、性別、角色、電話、Email、團別、年級、職位、隊名 |
| `家庭資料` | 家庭單位資訊 | 家庭ID、家庭名稱、家庭編號 |
| `活動列表` | 活動行程管理 | 活動ID、活動名稱、開始/結束日期、截止報名日期、狀態 |
| `報名紀錄` | 各家庭報名狀態 | 報名ID、活動ID、成員ID、出席狀態、備註 |
| `收費活動` | 收費項目主檔 | 活動ID、活動名稱、說明、類型 |
| `收費設備` | 設備/品項定義 | 設備ID、活動ID、設備名稱、金額、數量上限、種類 |
| `收費明細` | 各家庭選購明細 | 明細ID、活動ID、成員ID、家庭ID、設備ID、數量、單價、小計 |
| `繳費紀錄` | 繳費申報與審核 | 紀錄ID、家庭ID、繳費金額、繳費方式、狀態、退回原因 |
| `管理員帳號` | 登入憑證 | 帳號、密碼雜湊（MD5） |
| `對帳備註` | 管理員對帳備忘 | 活動ID、家庭ID、管理員備註 |

### 資料流程

```
家長開啟網頁
    │
    ├─ 一般頁面
    │   ├─ 選擇活動
    │   ├─ 輸入家庭 ID → 讀取「成員資料」
    │   ├─ 勾選成員出席狀態
    │   └─ 送出 → 寫入「報名紀錄」
    │
    └─ 管理員頁面（?page=admin）
        ├─ 登入驗證（MD5 比對「管理員帳號」）
        ├─ 成員/家庭管理 ↔ 「成員資料」「家庭資料」
        ├─ 活動管理 ↔ 「活動列表」
        ├─ 出席統計 ← 彙整「報名紀錄」+「成員資料」
        └─ 收費管理 ↔ 「收費活動」「繳費紀錄」
```

## 前置需求

- 一個 **Google 帳號**
- 一份 **Google Spreadsheet**（作為資料庫，記下試算表 ID）
- 一個 **Google Apps Script** 專案

> 無需本地開發環境，所有開發與部署皆在 Google 雲端完成。

## 安裝說明

### 步驟一：建立 Google Spreadsheet

1. 前往 [Google Sheets](https://sheets.google.com) 新建一份空白試算表
2. 記下網址列中的試算表 ID：
```
   https://docs.google.com/spreadsheets/d/【這段就是 ID】/edit
```

### 步驟二：建立 Google Apps Script 專案

1. 前往 [Google Apps Script](https://script.google.com) 新建專案
2. 依序建立所有 `.gs` 腳本檔案與 `.html` 前端檔案（見上方檔案結構）
3. 將本 Repository 對應檔案的完整內容貼入各檔案

### 步驟三：設定資料庫 ID

開啟 `Helpers.gs`，將試算表 ID 填入：

```javascript
var DATABASE_SHEET_ID = '【貼上您的試算表 ID】';
```

### 步驟四：初始化工作表

在 Apps Script 編輯器中，從函式下拉選單選取 `setupSheets`，按下執行（▶）。

> 首次執行時 Google 會要求授權，請允許所需的存取權限。

### 步驟五：部署為網頁應用程式

1. 點選右上角「部署」→「新增部署作業」
2. 類型選擇「**網頁應用程式**」
3. 設定：
   - **執行身分**：您自己（Me）
   - **具有存取權的使用者**：所有人（Anyone）
4. 點擊「部署」，複製並保存產生的網頁應用程式網址

## 使用說明

### 一般會員（家長）

1. 開啟網頁應用程式網址
2. 從活動列表選擇活動（僅顯示開放報名且未超過截止日期的活動）
3. 輸入家庭 ID 載入家庭成員名單
4. 為每位成員選擇出席狀態（出席 / 不出席 / 待定），可選填備註
5. 點擊「送出報名」；若需修改，重新填寫即可覆蓋原有紀錄

### 管理員

在網址後加上 `?page=admin` 進入管理員頁面：

```
https://script.google.com/.../exec?page=admin
```

| 功能模組 | 說明 |
|---------|------|
| **成員管理** | 新增、編輯、刪除成員資料（含角色、年級、隊別等） |
| **家庭管理** | 維護家庭單位資料 |
| **活動管理** | 建立/編輯活動，設定開放報名日期與截止日期 |
| **出席統計** | 依活動查看家庭/團別/小隊出席率 |
| **收費管理** | 建立收費活動與設備項目，審核家庭繳費申報 |

## 管理員帳號設定

系統以 **MD5 雜湊**保護管理員密碼，初次設定步驟：

1. 開啟 `Main.gs`，在 `generatePasswordHashTool` 函式中填入密碼：
```javascript
   var rawPassword = "your_password_here";
```
2. 執行函式，在「執行記錄」中複製 MD5 雜湊值
3. 在試算表「**管理員帳號**」工作表的 A 欄填入帳號，B 欄貼上雜湊值

> ⚠️ 請勿在試算表中直接儲存明碼密碼。
> 若工作表不存在，可先執行 `fixAdminSheet()` 函式自動建立。

## 常見問題

**Q：執行 `setupSheets` 後工作表沒有出現？**
確認 `Helpers.gs` 中的 `DATABASE_SHEET_ID` 是否正確，以及帳號是否有試算表編輯權限。

**Q：部署後開啟網頁出現「權限不足」？**
部署時「具有存取權的使用者」請設為「所有人（Anyone）」。

**Q：修改程式碼後如何更新？**
「部署」→「管理部署作業」→「編輯」→「建立新版本」→「部署」。

**Q：如何匯出報名或繳費資料？**
資料直接儲存於 Google Sheets，可在試算表中篩選後匯出為 Excel 或 CSV。

**Q：如何測試後端資料讀取是否正常？**
執行 `testGetAllData()` 函式，執行記錄會顯示各工作表的資料筆數。
